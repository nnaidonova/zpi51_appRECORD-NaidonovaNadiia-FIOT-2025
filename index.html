Лабораторна робота №3

Лабораторна робота №1: ТЕГИ ТА АТРИБУТИ HTML-ДОКУМЕНТА

1. Тема, мета, місце розташування

Тема: розробка семантичного HTML-документа веб-застосунку та використання тегів і атрибутів HTML5.

Мета: створити односторінковий сайт (landing page) вейк-школи WakeFlow з основними інформаційними розділами (про школу, послуги, команда, галерея, контакти) і формою запису на заняття. Форма збирає дані користувача та передає заявку до Google Sheets через Google Apps Script.

Місце розташування роботи:

  • Репозиторій з кодом: посилання на GitHub
  • Демо-сторінка (GitHub Pages): посилання на live
  • Сторінка «Бізнес-логіка» (business.html): посилання
  • Figma дизайн: відкрити макет

2. Предметна область та бізнес-логіка

2.1. Предметна область

Веб-застосунок WakeFlow представляє послуги вейк-школи: навчання вейкбордингу, оренду/підбір спорядження, презентацію команди інструкторів, фотогалерею та контактні дані. Користувач може залишити заявку на тренування через форму запису.

2.2. Основні сутності (дані)

  • Послуга: назва, тривалість, формат (індивідуально/група), ціна.
  • Інструктор: ім’я, фото, спеціалізація/роль, опис.
  • Галерея: зображення, опис/підпис (alt/figcaption).
  • Заявка: ПІБ, телефон, email, рівень підготовки, формат, дата, коментар, згода на обробку даних.

2.3. Ключовий бізнес-процес: «Запис на заняття»

  1. Користувач заповнює форму (контактні дані + деталі тренування) і підтверджує згоду.
  2. Після натискання кнопки «Надіслати заявку» дані відправляються методом POST до Google Apps Script.
  3. Google Apps Script зберігає запис у Google Sheets (таблиця заявок).
  4. Користувач отримує повідомлення про успішну відправку або помилку (мережа/доступ).

3. Структура HTML-документа (семантична верстка)

Головна сторінка WakeFlow реалізована як семантичний HTML5-документ із використанням структурних тегів: <header>, <nav>, <main>, <section>, <footer>. Для внутрішньої навігації застосовано якірні посилання на секції з ідентифікаторами.

Приклад загального «каркасу» сторінки:

<body>
  <header>
    <!-- логотип + навігація -->
  </header>

  <main>
    <section id="about">...</section>
    <section id="services">...</section>
    <section id="team">...</section>
    <section id="gallery">...</section>
    <section id="contact">...</section>
  </main>

  <footer>...</footer>
</body>

Навігація виконується через href="#..." до секцій: #about, #services, #team, #gallery, #contact.

4. Приклад гіперпосилань і навігації

Фрагмент шапки сайту з логотипом і меню навігації (атрибути: href, aria-label, id, alt, width):

<header>
  <a href="index.html">
    <img id="WakeFlow_logo" src="img/logo.png" alt="WakeFlow_logo" width="100" />
  </a>

  <nav aria-label="Головна навігація">
    <ul>
      <li><a href="#about">Про школу</a></li>
      <li><a href="#services">Послуги</a></li>
      <li><a href="#team">Команда</a></li>
      <li><a href="#gallery">Галерея</a></li>
      <li><a href="#contact">Контакти</a></li>
      <li><a href="business.html">Бізнес-логіка</a></li>
    </ul>
  </nav>
</header>

5. HTML-код таблиці

Для відображення прайсу застосовано таблицю з підписом (<caption>), заголовком (<thead>) і тілом (<tbody>), а також атрибутами доступності (aria-label, scope).

<section id="services" aria-labelledby="services-title">
  <h2 id="services-title">Послуги та ціни</h2>

  <div class="table-wrap" role="region" aria-label="Прайс-лист (прокручуваний блок)">
    <table aria-label="Таблиця послуг вейк-школи">
      <caption>Прайс-лист WakeFlow (демо)</caption>

      <thead>
        <tr>
          <th scope="col">Послуга</th>
          <th scope="col">Тривалість</th>
          <th scope="col">Для кого</th>
          <th scope="col">Ціна</th>
        </tr>
      </thead>

      <tbody>...</tbody>
    </table>
  </div>
</section>

6. HTML-код зображення

Зображення оформлено в семантичній парі <figure> + <figcaption>. Використано атрибути: src, alt, width.

<figure>
  <img src="img/hero.png"
       alt="Катання на вейкборді за катером (тестове зображення)"
       width="1200" />
  <figcaption>WakeFlow — школа водних видів спорту.</figcaption>
</figure>

7. HTML-код списків

Для структурованої подачі інформації застосовано маркований (<ul>) і нумерований (<ol>) списки.

<h3 id="advantages-title">Чому з нами зручно</h3>
<ul>
  <li>Інструктори поруч на кожному етапі навчання</li>
  <li>Пояснюємо техніку просто і по ділу</li>
  <li>Тестове спорядження для перших занять</li>
  <li>Фото/відео фрагменти прогресу (за можливості)</li>
</ul>

<h4>Як проходить перше заняття</h4>
<ol>
  <li>Короткий інструктаж з безпеки</li>
  <li>Підбір спорядження під зріст і рівень</li>
  <li>Перші старти з води та стабілізація стійки</li>
  <li>Розбір помилок та план наступного кроку</li>
</ol>

8. HTML-код форми

Форма реалізована через <form> з групуванням полів у <fieldset> і підписами <label>. Використані атрибути: id, name, type, required, method, aria-label. 

<form id="leadForm" action="#" method="post" aria-label="Форма запису на заняття">

  <fieldset>
    <legend>Контактні дані</legend>

    <label for="fullName">Ім’я та прізвище:</label>
    <input id="fullName" name="fullName" type="text" required />

    <label for="phone">Телефон:</label>
    <input id="phone" name="phone" type="tel" required />

    <label for="email">Email (необов’язково):</label>
    <input id="email" name="email" type="email" />
  </fieldset>

  <fieldset>
    <legend>Деталі заняття</legend>

    <select id="level" name="level" required>...</select>

    <input type="radio" name="format" value="Індивідуально" checked />
    <input type="radio" name="format" value="Мінігрупа" />

    <input id="date" name="date" type="date" />

    <textarea id="message" name="message"></textarea>

    <input type="checkbox" name="agree" required />
  </fieldset>

  <input type="hidden" name="source" value="WakeFlow site" />
  <input type="hidden" name="pageUrl" id="pageUrl" value="" />

  <button type="submit">Надіслати заявку</button>
</form>

10. Висновок

У ході лабораторної роботи №1 було створено семантичний HTML-документ веб-застосунку WakeFlow та застосовано основні HTML5-теги й атрибути для структурування контенту, навігації, таблиць, списків, зображень і форми введення даних. Реалізовано логіку введення даних користувача через форму запису, що є основою подальшого підключення бізнес-логіки та CSS-оформлення.

Лабораторна робота №2: CSS-оформлення

1. Тема, мета, місце розташування

Тема: стильове оформлення елементів HTML-документа засобами каскадних таблиць стилів (CSS). Використання селекторів тегу, класу, ідентифікатора, комбінаторів, псевдокласів.

Мета: набути практичних навичок підключення CSS до HTML-документа та застосування основних і розширених засобів CSS для оформлення шрифтів, тексту, таблиць, списків, фону/контурів, форм і адаптивного компонування сторінки.

Місце розташування роботи:

  • Репозиторій з кодом: GitHub
  • Демо-сторінка (GitHub Pages): live
  • Сторінка «Бізнес-логіка» (business.html): відкрити
  • Figma дизайн: відкрити макет

Файли проєкту:

  • index.html — головна сторінка застосунку WakeFlow
  • style.css — файл каскадних таблиць стилів

2. Способи підключення стилів

У проєкті WakeFlow використано підключення стилів через зовнішній CSS-файл (рекомендований спосіб), який підключається у секції <head> HTML-документа.

2.1. Зовнішній CSS-файл (використано у проєкті)

<link rel="stylesheet" href="style.css" />

2.2. Внутрішні стилі (приклад)

<style>
  body { font-family: system-ui, Arial, sans-serif; }
</style>

2.3. Інлайн-стилі (приклад)

<p style="color:#fff; margin:0;">Текст</p>

Основний підхід у роботі — централізоване керування оформленням через style.css, що спрощує підтримку та масштабування інтерфейсу.

3. СЕЛЕКТОРИ

Для стилізації елементів сторінки застосовано селектори різних типів: тегів, класів, ідентифікаторів, а також комбіновані селектори, селектори атрибутів та псевдокласи. Нижче наведено приклади з файлу style.css.

3.1. Селектори тегу

Селектори тегів використовуються для базового оформлення елементів документа.

body{
  margin:0;
  line-height: 1.55;
  color: var(--text);
}

img{
  display:block;
  max-width:100%;
  height:auto;
}

table{
  width:100%;
  border-collapse: collapse;
  min-width: 720px;
}

th, td{
  padding: 12px;
  border-top: 1px solid var(--border);
}

3.2. Селектори класу

Класи застосовуються для багаторазових компонентів інтерфейсу (кнопки, сітки, статуси).

.btn{
  display:inline-flex;
  align-items:center;
  justify-content:center;
  padding: 11px 14px;
  border-radius: 999px;
  border: 1px solid var(--border);
  background: rgba(255,255,255,.06);
  color: var(--text);
}

.gallery-grid{
  display:grid;
  grid-template-columns: repeat(3, 1fr);
  gap: 12px;
  margin-top: 12px;
}

.status.success{
  border-color: rgba(182,255,59,.35);
  background: rgba(182,255,59,.10);
}

HTML-приклад використання класу:

<a class="btn primary" href="#services">Переглянути послуги</a>

3.3. Селектори ідентифікаторів

ID-селектори застосовуються для унікальних елементів сторінки (наприклад, логотип).

#WakeFlow_logo{
  border-radius: 100%;
  background: transparent;
}

HTML-приклад використання id:

<img id="WakeFlow_logo" src="img/logo.png" alt="WakeFlow_logo" width="100" />

3.4. Інші селектори

Універсальний селектор

*{ box-sizing: border-box; }

Дочірній комбінатор (child combinator)

header > div a{
  display:inline-flex;
  align-items:center;
  gap: 12px;
}

Селектор атрибуту (attribute selector)

nav[aria-label="Головна навігація"] ul{
  list-style:none;
  display:flex;
  flex-wrap:wrap;
  gap:10px;
}

Псевдокласи

a:hover{ color: var(--accent); }

input:focus, select:focus, textarea:focus{
  border-color: rgba(70,215,255,.55);
  box-shadow: 0 0 0 4px rgba(70,215,255,.12);
}

nav[aria-label="Головна навігація"] li:last-child a{
  border-color: rgba(182,255,59,.35);
  background: rgba(182,255,59,.12);
}

Додатково (за потреби приклад “сусіднього селектора”):

/* Adjacent sibling selector (для ЛР2): абзац одразу після h3 */
h3 + p{ margin-top: 6px; }

4. CSS: Шрифти, текст, таблиці, фон, контур, списки, CSS просунутий

4.1. Шрифти і текст

body{
  font-family: system-ui,-apple-system,Segoe UI,Roboto,Arial,sans-serif;

6. Висновки

У ході виконання лабораторної роботи №2 реалізовано CSS-оформлення веб-застосунку WakeFlow із використанням зовнішнього файлу стилів. Опрацьовано базові принципи каскадування та спадкування, а також застосовано змінні CSS для уніфікації палітри, відступів і візуальних параметрів інтерфейсу.

У роботі використано різні типи селекторів: селектори тегів, класів, ідентифікаторів, атрибутів, комбінатори та псевдокласи для інтерактивних станів елементів. Для компонування сторінки застосовано Flexbox і CSS Grid, що забезпечило структурованість і масштабованість макета.

Реалізовано оформлення шрифтів і тексту, таблиць, списків, фону та контурів, а також адаптивні правила через медіа-запити для коректного відображення на різних пристроях. Отримані навички дозволяють створювати підтримуване та візуально узгоджене CSS-оформлення веб-сторінок.
